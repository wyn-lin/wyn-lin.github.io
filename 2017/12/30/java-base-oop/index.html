java 面向对象基础

发表于 2017-12-30 | 分类于 java基础

java用引用操纵对象

在java里，一切都被当做对象，操纵对象的标识符实际上是对象的一个引用，可以想象成一个遥控器和一个电视机，拥有一个引用并不一定需要有个对象与之关联，引用没有对象与之关联的话就是为null

对象在内存中的存储

  • 寄存器

最快的存储区，数量有限，根据需求进行分配，不能够直接控制

  • 堆栈

一种随机访问存储器，对象的引用存放于此

  • 堆

通用内存区也位于内存，用于存放所有的java对象

  • 常量存储区

常量通常直接存放在代码内部

  • 非ram存储器

存活于程序之外，持久化对象，对象被存放在磁盘，把对象转化成可以存放在其他媒介上的事物，在需要的时候可恢复成内存中存在的对象，java提供了轻量持久化的支持。

java作用域

c++中允许在小的作用域内定义名称一样的变量，但是在java中是不允许的，但是方法的形参例外。

基本数据类型变量的生命周期在其作用域内，而引用类型的对象可以存活于作用域之外

字段和方法

基本数据类型的局部变量不会自动赋初始值，引用类型的局部变量默认为null
